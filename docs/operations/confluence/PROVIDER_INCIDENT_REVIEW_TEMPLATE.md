# Provider Incident Review Template

## Canonical repo sources

- `docs/operations/JIRA_PROVIDER_OPS.md`
- relevant Jira `PROVOPS` incident issues

## Metadata

- Incident title
- Provider family
- Date
- Owner
- Jira incident link
- Follow-up Jira links

## Timeline

- first signal detected
- impact window
- mitigations applied
- recovery time

## Trigger and symptoms

- what failed
- what user-facing symptoms appeared
- what routing, health, or billing signal triggered response

## Root cause

- technical cause
- contributing factors
- why existing safeguards did or did not help

## Remediation

- immediate fix
- longer-term fix
- docs/process changes

## Follow-up

- owner
- due date
- linked `PROVOPS` or `PLAT` tickets
