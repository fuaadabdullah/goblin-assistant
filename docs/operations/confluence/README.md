# Confluence Page Sources

These files are repo-tracked source material for the `Provider Architecture` Confluence space.

They are not the canonical source for runtime config or provider adapter behavior. Their purpose is to:

- provide stable page structure for Confluence
- keep Confluence page intent reviewable in git
- reduce copy/paste drift across architecture, onboarding, and incident pages

Source hierarchy:

- runtime/provider truth: `config/providers.toml`
- adapter and onboarding truth: `apps/api/src/api/providers/README.md`
- operational/provider matrix truth: `docs/operations/PROVIDERS.md`
- Confluence page source material: files in this directory
